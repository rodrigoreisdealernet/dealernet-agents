# Marketing Plan - 2026-06-19

Today's story is clearer operational decision-making across the rental day. Teams now get stronger queues, more actionable review surfaces, and more readable screens so they can move from triage to follow-up with less reconstruction work.

## Start the day from one technician queue
- **Promotes:** Start the day from one technician queue - Shipped in #2230
- **Audience / persona:** Branch managers and field operators who need to sort returned units, PM work, repairs, and rent-ready checks into the right order each morning
- **Value proposition:** Give service teams one ranked queue so they can start the day faster and spend less time debating what should come first.
- **Talking points:**
  - Bring returned units, PM work, repairs, and rent-ready checks into one morning decision point.
  - Show why each item is urgent so crews can make faster calls with less back-and-forth.
  - Preserve human priority overrides when the branch needs to adapt to local conditions.
- **Suggested channels:** in-app announcement, customer email, LinkedIn/social, changelog line

- **In-app announcement** (<160 chars): "Start the day with one ranked technician queue for returns, PM work, repairs, and rent-ready checks."
- **Email blurb** (subject + 2-3 sentences): "Subject: New morning technician queue for service teams. Branches can now start the day from one ranked queue that brings returned units, PM work, repairs, and rent-ready checks together in one place. Each item explains why it is urgent, with room for human priority overrides when local conditions change."
- **Social / LinkedIn post** (<1-2 short paragraphs + CTA): "Morning shop planning works better when every team starts from the same queue. - **Changelog line** (one sentence): "Service teams now get one ranked morning technician queue for returns, PM work, repairs, and rent-ready checks, with urgency reasons and human overrides."
