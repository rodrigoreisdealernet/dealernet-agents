# Marketing Plan - 2026-06-20

Today's story is more confident operational follow-through. Teams can now document work, approve units, run compliance reviews, and move from recommendation to action with less context loss and fewer hidden gaps.

## Document repairs from the work order itself
- **Promotes:** Document repairs without leaving the work order - Shipped in #2232
- **Audience / persona:** Branch managers and field operators who need to close repair work cleanly while the service history and current fault context are still in view
- **Value proposition:** Help technicians finish repair documentation, status updates, and escalation decisions from one work-order screen instead of stitching the story together across tabs.
- **Talking points:**
  - Bring recent faults, service history, and repair documentation into the same work-order flow.
  - Reduce screen switching when technicians need to update status or escalate a job.
  - Keep documentation closer to the actual repair moment, which supports cleaner handoffs.
- **Suggested channels:** in-app announcement, customer email, LinkedIn/social, changelog line

### Ready-to-use copy
- **In-app announcement** (<160 chars): "Document repairs, update status, and escalate work from the work order with recent faults and service history already in view."
- **Email blurb** (subject + 2-3 sentences): "Subject: Repair documentation now stays inside the work order. Service teams can now document repairs, update work-order status, and escalate from one screen with recent faults and service history already in view. It gives technicians a straighter path from doing the work to recording it clearly."
- **Social / LinkedIn post** (<1-2 short paragraphs + CTA): "Repair follow-through works better when technicians do not have to leave the work order to finish the record. This release adds a repair documentation copilot that brings recent faults and service history forward, then helps teams document the repair, update status, or escalate from the same screen.\n\nIf your service teams want cleaner handoffs with less screen switching, review today's docs link in the release notes."
- **Changelog line** (one sentence): "Service teams can now document repairs, update work-order status, and escalate from the work-order detail page with recent faults and service history in view."

- **Call to action:** Open the work-order detail page on the next repair and complete the documentation flow without leaving the screen.
- **Learn more:** #2423 (guide in progress)
