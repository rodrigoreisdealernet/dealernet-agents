The capability map for this vertical: what it takes to operate the business, decomposed
into capability areas → roles → real tasks. Roles live in [`roles/`](./roles/); coverage is
reported by `operating-model.ts coverage`. Every task must cite evidence.

This file is the **initial decomposition** — the structural answer to the north-star
question. The *empirical* layer (the specific tasks each role performs, with cadence and
pain) is populated by the [`domain-cartographer`](../../../../.github/agents/domain-cartographer.agent.md)
from cited evidence; **the decomposition here is design, the tasks underneath are evidence.**

## Capability areas

The major functional areas it takes to run the business. Each maps to one or more roles;
each is a lens for the cartographer's task hunt and a frame for the discovery roadmap.

| Capability area | What it covers |
|-----------------|----------------|
| **Fleet & asset management** | Acquiring, tracking, valuing, and disposing of the rental fleet; utilization and ROI |
| **Rental lifecycle** | Quote → reservation → contract → checkout → on-rent → return → off-rent billing |
| **Logistics & transport** | Delivery/pickup scheduling, routing, yard staging, driver/ELD compliance |
| **Service & maintenance** | Preventive maintenance, repairs, inspections, damage assessment, parts |
| **Branch operations** | Day-to-day running of a branch: staffing, counter, safety, local P&L |
| **Sales & CRM** | Account management, quoting, contractor/project relationships, win/loss |
| **Procurement & re-rent** | Buying fleet & parts, vendor management, sourcing re-rent when short |
| **Billing, credit & finance** | Invoicing, rate plans, credit decisions, collections, revenue recognition |
| **Customer self-service** | External customer access to rental status, invoices, payments, pickup requests, and service requests |
| **Safety & compliance** | Operator safety, regulatory (DOT/ELD/environmental), certifications, audits |
| **Systems administration & integration** | User permissions, branch hierarchy, billing/report configuration, and cross-system data health |
| **Multi-branch & enterprise** | Cross-branch transfers, central fleet sourcing, regional/national rollups, reporting |
